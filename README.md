Vision statement: The website provides a list of courses. Users (students) can browse course reviews and write reviews for the course they have taken. They just need to fill up a pre-defined questionnaire to answer a few questions. The website will collect, organize, and display their reviews. The website will have a login system. Students can create anonymous (regular) accounts. The website will also have administrator accounts, which will be used for moderation and site administration (add new courses, delete/flag posts). 

Motivation: We are working on this project to create a platform where students can provide honest feedback about courses within the CSPB program. This will allow students to read course reviews from their peers and make more informed decisions when selecting classes.

Risks to project completion, possibly including:
- Some team members have not taken a course involving SQL or have limited experience with SQL.
- This is a newly formed group, so we are still becoming familiar with each other's coding styles, experience levels, and time zone differences.
- Time constraints due to personal obligations, work commitments, and requirements from other courses.
- The scope of the project may be larger than what can realistically be completed within the course timeline.

Mitigation Strategy for above risks: To mitigate these risks, we discussed the project proposal and identified a minimum viable product (MVP) that will provide the core functionality of the website. By focusing on the MVP first, we can ensure that the essential features are completed and functioning properly. Through Scrum practices and weekly team meetings, we will monitor progress, address challenges, and determine whether additional enhancements to the site's functionality and visual design can be implemented. Our primary goal is to work collaboratively and deliver a functional, reliable website by the end of the project.
